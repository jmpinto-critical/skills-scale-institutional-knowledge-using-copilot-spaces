# OctoAcme Project Management Docs

Welcome! This README provides a brief overview of the project management processes used by OctoAcme, with links to all core documentation in the `docs/` directory. Use this as your starting point for understanding how OctoAcme plans, delivers, and continuously improves its projects.

---

## Summary of Project Management Processes

OctoAcme uses an **iterative, customer-led approach** focused on clear ownership, strong collaboration, quality delivery, and continuous improvement.

### Principles
- **Customer-first**: Prioritize customer value and usability in every decision.
- **Iterative delivery**: Deliver small, testable increments to reduce risk and increase feedback.
- **Clear ownership**: Each project has a named Project Manager (PM) and Product Lead.
- **Data-informed decisions**: Measure impact and iterate based on evidence.
- **Psychological safety**: Encourage open feedback and a culture of learning.

### Project Lifecycle
Projects follow five key stages:

1. **Initiation** — Validate the business need, align stakeholders, and produce a Project One-pager with goals, success metrics, and a high-level timeline.
2. **Planning** — Break work into shippable increments, define the Definition of Done (DoD), estimate scope, map dependencies, and create a release plan.
3. **Execution & Tracking** — Build and test in sprints using GitHub Projects. Run daily standups, weekly delivery syncs, and end-of-sprint demos. Escalate blockers through a defined path.
4. **Risk Management & Communication** — Maintain a Risk Register, provide regular stakeholder updates, and follow clear escalation paths for issues and incidents.
5. **Release & Deployment** — Follow a standardized deployment checklist, run smoke tests, and communicate releases to stakeholders and support.
6. **Retrospective & Continuous Improvement** — After each sprint or milestone, capture what went well, what can be improved, and convert learnings into backlog action items.

### Core Roles
| Role | Responsibility |
|---|---|
| **Project Manager (PM)** | Coordinates delivery, schedules, risk, and communications |
| **Product Manager (PdM)** | Defines outcomes, prioritizes backlog, and measures success |
| **Developers** | Implement features, collaborate on design and testability |
| **QA/Testing** | Validate quality and acceptance criteria |
| **Stakeholders** | Provide inputs and approvals |

### Key Artifacts
- Project Charter / One-pager
- Roadmap and Release Plan
- Sprint/Iteration Backlog
- Acceptance Criteria & Definition of Done
- Risk Register
- Retrospective Notes and Action Items

### Communication Cadence
- Weekly sync between PM + PdM
- Twice-weekly standups for the delivery team (or as agreed)
- Monthly stakeholder updates
- Ad-hoc escalations as needed
